# WatchBuddy 手表宠物格式

`watch-pet.json` 是 WatchBuddy 构建期转换器产生的可信清单，不是对远程宠物包开放的上传格式。
手表只读取经过源格式、授权、图片和摘要校验的转换结果。

## 处理边界

```text
Codex Pet v2 源包
  -> 完整校验 pet.json 与 1536×2288 图集
  -> 固定来源和 SHA-256
  -> 裁帧、缩放、压缩
  -> 生成 watch-pet.json
  -> 再次校验清单和文件
  -> 打入 HAP 或进入受控资源目录
```

源包必须是 `spriteVersionNumber: 2`。8 × 9 旧包只能进入显式迁移流程，不能静默标记为 v2。

## 强制限制

| 项目 | 限制 |
|---|---:|
| 清单版本 | `schemaVersion: 1` |
| 渲染器 | `frame-sequence-v1` |
| 资源格式 | 本地 PNG/WebP |
| 单帧上限 | 64 KiB |
| 总资源上限 | 2 MiB |
| 资源数量上限 | 88 |
| 原始来源 | 不含凭据的 HTTPS URL |
| 授权 | 必须明确允许再分发 |
| 摘要 | 源文件与每个输出资源都使用小写 SHA-256 |

清单禁止绝对路径、URL 资源、路径穿越、未知字段、未知动画和未知许可证。所有动画帧、
注视方向、状态映射、交互映射和降级帧都必须引用清单内的资源。

JSON Schema 负责结构约束，`validateWatchPetManifest` 额外检查引用、唯一性、派生预算和安全语义。
